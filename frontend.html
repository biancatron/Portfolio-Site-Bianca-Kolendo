Front-end Development

In one month’s time, develop a portfolio site to be able to present my UX Design work. This project gave me an introduction into HTML, CSS, and JS. Following a design process, the site was tested with participants for improvements. To ensure clean code, all pages were linted and checked against the most common browsers.

Context + Challenge

Project Brief

Code a portfolio site for future employers to see my design work. I worked with HTML, CSS, and JS to create my portfolio.

Methods + Tools

  • HTML, CSS, and JS in Atom
  • Sourcecode management with GitHub
  • Remote moderated user testing
  • Browser cross testing with BrowserStack

Timeline

May 10, 2021 - June 3, 2021

The Problem

As a UX Designer, I need a way to present my talent, beyond my current resume and cover letter. By learning about frontend development, the process and limitations, I can better collaborate and communicate with developers.

Goals

Code a responsive website that can be viewed in any situation. I would like to track visits as well as downloads of my resume.

Process + Insights

The first step in building my portfolio was to plan the content. This includes gathering all my content for the case studies, deciding on an accessible color palette, and picking fonts for each piece of content.

Research

Layouts for the pages were provided by CareerFoundry. Along with the bootcamp, I used Codecademy + W3schools to practice and learn more about CSS + JS.


By starting with the foundational HTML coding, I was able to quickly add all my content to the site. This made it easy to add style and interactivity all at once with CSS + JS. I even played around with animating an SVG!

Testing

To help me identify errors throughout my site, I recruited 5 participants for remote moderated user testing. The participants were all involved in the hiring process at some capacity, either on an interview panel or the hiring manager.
They had to complete 4 scenario tasks to test the functionality of my site in mobile, tablet, or desktop view.

  • Review the homepage
  • Locate information on how I handle usability testing in any case study
  • Find my resume
  • Send me a message

"These are your accomplishments, but they’re all very different images. How do these all connect?”

It was exciting for me to see people use a site I had coded myself! Unsurprisingly, the site had many errors. The most problematic issue was the competing navigation bars. My header was stuck behind my side navigation making it impossible to navigate anywhere but the homepage. I also got great feedback on the visual aspects of my content. For example, although two of my projects were app based case studies, the homepage images used don’t communicate that.

Errors + Improvements

01: On case study pages in mobile view, the side navigation will block the opened hamburger menu.

High Severity
Recommendation:
Lay hamburger menu options on top of side nav bar//Add more top padding to side nav when opening hamburger menu.

02: The side nav on project pages overlaps the footer on mobile view.

High Severity
Recommendation:
Add left padding to the footer on project pages.

03: Selecting a topic in the side nav jumps to the anchor on the page, but the start of the anchor is below the header.

Medium Severity
Recommendation:
Add margins to the headings, which are used as anchor points.

04: Selecting a topic in the side nav jumps to the anchor on the page, but the start of the anchor is below the header.

Low Severity
Recommendation:
Add left/right padding in buttons.

05: The “!” is outside of the ‘send me an email’ button.

Low Severity
Recommendation:
Move the “!” into the button.

Browser Cross Testing

The first set of errors I addressed were identified through user testing. Once I was able to update the majority of those, I needed to ensure my site would be functional on any device, and any browser. I conducted Browser Cross Testing with BrowserStack and found code issues with Safari.



Conclusion

I was able to create a responsive portfolio site, with layout variations for mobile, tablet, and desktop views. The site has a simple design to help with quick navigation and features additional interactivity only when needed. For example, the side navigation used on case study pages as they contain a lot of content. The color palette is in compliance with WCAG AAA standards, but was also selected to reflect my personality. The yellow header is fun and playful, and the deep teal is calming and trustworthy. Overall, I believe this is a great start to helping me introduce myself to hiring managers.
